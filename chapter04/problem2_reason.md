## 현재 코드를 짠 이유
- 나이트의 이동 법칙에 의하면 최대로 나올 수 있는 경우의 수는 총 8가지이므로 Brute Force 방식으로 문제를 해결할 수 있다고 생각하여 진행했다.

### 풀이의 코드와 나의 코드가 다른 점  
1. input   
   -  python의 input() 함수를 통해서 사용자의 입력을 받아들이면 String 타입으로 들어오기 때문에 str(input())은 무의미하다.   
    - String은 자체적으로 특정 위치의 요소를 가져올 수 있기 때문에 list(str(input())) 또한 무의미하다.
2. 나이트의 이동 방향   
   - 조건에서 제시한 나이트의 이동 방향을 하나로 합칠 수 있다.
3. 단일 for 문   
    - 2번의 차이점에서 나이트의 이동 방향을 하나로 합치게 되면 for문도 2개를 사용할 필요 없이 하나만 사용할 수 있다.
4. 체스판을 넘었는지 안넘었는지에 대한 조건문   
    - 본인의 코드는 체스판을 넘긴 경우일 때 다시 loop를 돌게 설정하였지만 해설의 코드에서 처럼 체스판을 넘지 않은 경우만 count를 하게 되면 코드가 더 깔끔해진다.
